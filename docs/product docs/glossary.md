# Глоссарий

**Версия:** 0.8  
**Дата:** 2026-03-31  
**Назначение:** единые определения терминов для `metrics-registry`, `data-map`, `prd`, `dev-handoff-spec` и обсуждений с бизнесом.  
**Правило:** новый термин, который попадает в реестр метрик или справочники — по возможности сначала кратко фиксировать здесь.

---

## Как вести документ

- Новые строки — в логическую секцию; при появлении **ID метрик** в реестре — дублировать ссылку на ID в примечании.  
- **Каналы** и **точки** — только из согласованных справочников (ниже).  
- Термины вроде «лид» / «сделка» в разговоре допустимы; в формах v1 использовать те же поля, что в `metrics-registry.md`, без обязательной унификации жаргона, пока не появится CRM-контур.

## Справочник точек v1 (физические точки)

Норматив для сида БД, API и UI. Коды **латиницей**, стабильны; менять только через явную миграцию / согласование с владельцем продукта. Техническая модель — [`architecture-draft.md`](../development%20docs/architecture-draft.md) §4.1.

| Код | Отображаемое имя |
|-----|------------------|
| `NOVOGRAD` | Точка на Новоградском |
| `SVERDLOV` | Точка на Свердловском |

## Справочник каналов (закрытый список)

Использовать в метриках и формах как нормализованное значение «канал» (точное написание для разработки можно зафиксировать латиницей в `data-map.md`).

| Канал (рабочее название) | Комментарий |
|---------------------------|-------------|
| Сайт | Органика / сайт компании |
| Контекстная реклама | Платный контекст |
| Прямые обращения | Постоянные клиенты, звонки и прочие прямые контакты без отнесения к другому каналу |
| 2ГИС | Картографический сервис |
| Яндекс.Карты | Картографический сервис |
| **Ozon** | **Отдельный канал** продаж (маркетплейс) и одновременно **отдельная виртуальная точка** в продукте: по нему есть **своя выручка** и тот же набор полей ввода/аналитики, что у офлайн-точек (`OZ-*`, см. `metrics-registry.md` §4 и правило **Ozon** в реестре). **Не** относить к **двум физическим** точкам справочника |
| Авито | Ранее использовался; **сейчас не используется** — в вводе v1 по умолчанию не активен, пока бизнес не вернёт канал |

## Продажи и деньги (определения для дашборда)

| Термин | Определение | Примечание / синонимы |
|--------|-------------|------------------------|
| Выручка | Денежная сумма, фактически в кассе или поступившая на счёт (по согласованной методике периода) | **Без вычета возвратов** в продукте; не «чистая выручка» |
| Заказ | Одна совершённая покупка / один заказ в учёте точки | Один заказ = единица числителя для среднего чека |
| Средний чек | Выручка / количество заказов за тот же период и разрез | Производный показатель |
| Возврат (в дашборде) | Отдельные поля за отчётную неделю: **по офлайн-точке** (`OFF-RET-*`) и в Ozon (`OZ-RET-*`) | **Не** уменьшают выручку в формулах продукта; как сопоставлять с продажами в учёте компании — **не детализируется в v1** |
| Чистая выручка | Выручка за вычетом возвратов и т.п. | **Нет** в v1; не закладывать формулы и PRD под это |

## Продукт и данные

| Термин | Определение | Примечание / синонимы |
|--------|-------------|------------------------|
| Отчётная неделя | Календарная неделя **пн–вс**, данные вносятся за **прошлую** неделю (ритм: по понедельникам) | Ключевая гранулярность MVP |
| Snapshot (снимок) | Значение накопительного показателя **на дату ввода** (например, число отзывов) | Прирост считает **система**, не ручная дельта |
| Первичное поле | Показатель, который вводит пользователь в форме | См. колонки в `metrics-registry.md` |
| Производный показатель | Рассчитывается системой из первичных и/или прошлых снимков | Формулы — в реестре |
| Слой правды | Единое согласованное хранилище метрик для управленческих решений | Цель дашборда |
| Полнота недели | Состояние, когда для выбранной отчётной недели заполнены все обязательные поля ввода (по реестру), **включая нули** там, где нет факта | В v1 баннер полноты в UI нет — `prd.md` |

## Сайт и Яндекс.Метрика (v1)

| Термин | Определение | Примечание / ID |
|--------|-------------|-----------------|
| Посетители по каналам | Число **посетителей** за отчётную неделю в разрезе трёх фиксированных строк формы | `WEB-TRF-CH`, `WEB-TRF-TOT`; **не** сеансы |
| Каналы трафика сайта (форма) | **Органика**, **CPC (Директ)**, **Прямые заходы** — закрытый набор v1; сопоставление с группировкой Метрики — `data-map.md` | Отличается от «справочника каналов» компании (сайт, Директ, …) — это строки отчёта по сайту |
| Отказы / длительность (сайт) | Показатели Метрики на уровне **визита (сеанса)** за неделю **по сайту в целом**, без разбивки по каналам в v1 | `WEB-BEH-BOUNCE`, `WEB-BEH-TIME` (секунды) |
