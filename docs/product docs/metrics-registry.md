## Правила ведения реестра

| Правило | Содержание |
|---------|------------|
| **ID** | Стабильный код для ссылок из PRD, `data-map`, ТЗ; менять только при явной миграции. |
| **Тип** | Первичный — ввод в форме; Производный — считает система; Снимок — значение «на дату» для накопительных/моментных шкал. |
| **Гранулярность** | По умолчанию **отчётная неделя (пн–вс)**. Ввод по регламенту в **пн или вт** за **предыдущую** неделю — см. `glossary.md`. |
| **Исключение** | **Репутация на картах:** гранулярность **дата снимка**; **регламент ввода — понедельник** (в одном проходе с недельным контуром за предыдущую пн–вс); на графиках — по **дате ввода/снимка**; несколько снимков в одной календарной неделе в v1 не требуются. |
| **Единицы** | Деньги: **рубли с копейками**. Заказы, возвраты (шт), количество отзывов: **целые**. Средняя оценка на картах: **дробное число**. |
| **Точки офлайн** | Разрез — **2 физические точки** v1: коды **`NOVOGRAD`**, **`SVERDLOV`** (отображаемые имена — `glossary.md`, **Справочник точек v1**). |
| **Ozon** | **Канал продаж** (маркетплейс) и **отдельная виртуальная точка** в продукте: по нему есть **своя выручка** (`OZ-REV` и связанные `OZ-*`), поэтому блок ввода и своды по смыслу совпадают с офлайн-точками (заказы, возвраты, реклама); **не** входит в справочник **двух** физических точек. См. `glossary.md` — канал **Ozon**. |
| **Сравнения периодов** | **Год к году (YoY)** в v1 **нет**. На **главном дашборде владельца** для **всех блоков** сводки (**Сайт**, **Точки**, **Карты**, **Ozon**, **Возвраты**) — у ключевых KPI значения за **предыдущий период того же типа** (неделя/месяц/квартал) и **процент изменения**, где осмысленно (`prd.md`). Для **репутации** в месяце/квартале сравниваются агрегаты «последний снимок в периоде» между соседними периодами (§3). На **drill-down** и прочих графиках — в основном выбранный период; детали — [`dev-handoff-spec.md`](../development%20docs/dev-handoff-spec.md). Месяц/квартал для недельных метрик — **агрегаты из недель** (кроме репутации — §3). |
| **Возвраты vs выручка** | Возвраты — **отдельные** показатели. **Не** уменьшают `OFF-REV` / `OZ-REV`, **нет** «чистой выручки» и **нет** отката продаж в расчёте выручки в системе. |
| **Вне детализации v1** | Ввод и хранение возвратов — **по точке** (`OFF-RET-*`) и в Ozon (`OZ-RET-*`), за отчётную неделю; **не** вычитаются из выручки. Дополнительные правила (сопоставление с датой продажи, бухполитика, пересечение недель) продукт **на старте не детализирует** — по мере необходимости после запуска. |
| **Статус** | Все перечисленные ниже строки — **MVP**, если не указано иное. |

## 1. Продажи и возвраты — офлайн-точки

**Выручка (согласовано):** сумма денег, фактически попавших **в кассу или на расчётный счёт** за **календарную отчётную неделю** (пн–вс), по данным учёта точки — **только от продаж**, без вычета возвратов. Нал/безнал отдельно не разбиваем.

**Возвраты:** количество и сумма за ту же отчётную неделю — **всегда в разрезе офлайн-точки**; **не** пересчитывают `OFF-REV` и **не** используются как «откат» выручки в формулах продукта. Тонкости учёта на стороне бизнеса — см. правило **Вне детализации v1** выше.

| ID | Показатель | Тип | Первичный ввод / расчёт | Единица | Разрезы | Статус | Примечания |
|----|------------|-----|-------------------------|---------|---------|--------|------------|
| OFF-REV | Выручка | Первичный | Руководитель точки | руб. (копейки) | Точка (справочник) | MVP | За отчётную неделю; без вычета возвратов |
| OFF-ORD | Заказы | Первичный | Руководитель точки | шт (целое) | Точка | MVP | За отчётную неделю |
| OFF-RET-N | Возвраты, количество | Первичный | Руководитель точки | шт (целое) | Точка | MVP | За неделю; не влияет на OFF-REV |
| OFF-RET-SUM | Возвраты, сумма | Первичный | Руководитель точки | руб. (копейки) | Точка | MVP | За неделю; не влияет на OFF-REV |

### 1.1 Производные (считает система)

| ID | Показатель | Формула / логика | Зависимости | Статус |
|----|------------|------------------|-------------|--------|
| OFF-AVG-CHK | Средний чек | `OFF-REV / OFF-ORD` при `OFF-ORD > 0` | OFF-REV, OFF-ORD | MVP |
| OFF-RET-AVG | Средний чек возврата | `OFF-RET-SUM / OFF-RET-N` при `OFF-RET-N > 0` | OFF-RET-SUM, OFF-RET-N | MVP |

## 2. Реклама и сайт (ввод: Николай)

### 2.0 Расходы на рекламу (еженедельно, руб. с копейками)

В v1 **только суммы расходов**; **без разбивки по кампаниям**. В поле контекста в v1 учитывается **только Яндекс Директ**; иные контекстные площадки — при появлении — v2+ (см. `data-map.md`).

| ID | Показатель | Канал / носитель | Первичный ввод / расчёт | Единица | Статус | Примечания |
|----|------------|------------------|-------------------------|---------|--------|------------|
| MKT-AD-CTX | Расход на контекстную рекламу | Яндекс Директ | Первичный, Николай | руб. | MVP | В v1 — **только Директ**, одной суммой за отчётную неделю |
| MKT-AD-MAP | Расход на рекламу карточек на картах | 2ГИС / Яндекс.Карты (рекламные продукты для карточек) | Первичный, Николай | руб. | MVP | Одна сумма на неделю по **всем** картографическим сервисам; при появлении потребности в раздельном учёте — вынести вторую строку в v2 |
| OZ-AD-SPEND | Расход на рекламу на Ozon | Ozon | Первичный, Николай | руб. | MVP | Дублирует логическую связь с блоком Ozon; **одно поле в UI** — см. §4 |

*Итого расходов на рекламу за неделю — производный `DER-AD-TOTAL` (§5).*

### 2.1 Сайт: трафик по каналам и поведение (без денежной выручки)

Деньги с сайта **не оцениваем** в v1. Трафик по каналам — **посетители** (не сеансы) из **Яндекс.Метрики** за отчётную неделю; **закрытый набор из трёх строк** в форме и сопоставление с отчётом Метрики — в `data-map.md`. Показатели **отказов** и **средней длительности** в Метрике считаются на уровне **визита (сеанса)**; в продукте — **одна пара значений на сайт в целом** за ту же неделю (без разбивки по каналам трафика в v1).

| ID | Показатель | Тип | Первичный ввод / расчёт | Единица | Статус | Примечания |
|----|------------|-----|-------------------------|---------|--------|------------|
| WEB-TRF-CH | Посетители по каналам трафика | Первичный | Николай; три строки: Органика, CPC (Директ), Прямые заходы → посетители за неделю | посетители (целое) на канал | MVP | Источник и сопоставление с Метрикой — `data-map.md` |
| WEB-BEH-BOUNCE | Поведение: отказы (по сайту) | Первичный | Николай (из Метрики) | % (дробное) | MVP | За ту же отчётную неделю; метрика уровня визита в Метрике |
| WEB-BEH-TIME | Поведение: средняя длительность визита | Первичный | Николай (из Метрики) | сек (дробное) | MVP | За ту же неделю; **одно поле**; метрика уровня визита в Метрике |

## 3. Репутация (2ГИС, Яндекс.Карты)

**Модель данных:** снимок на **дату**; **одна карточка организации на точку** на каждой площадке. **Регламент:** ввод **по понедельникам**, в одном проходе с остальным недельным вводом за **предыдущую** пн–вс; значения снимаются **с экрана мобильного приложения** площадки (кабинет не обязателен). Визуализация по **оси «дата ввода/дата снимка»** (см. `data-map.md`).

| ID | Показатель | Площадка | Тип | Первичный ввод / расчёт | Единица | Разрезы | Статус | Примечания |
|----|------------|----------|-----|-------------------------|---------|---------|--------|------------|
| REP-RATING | Средняя оценка | 2ГИС / Яндекс.Карты | Снимок | Николай | дробное | Точка × площадка | MVP | На дату снимка |
| REP-REV-CNT | Количество отзывов | 2ГИС / Яндекс.Карты | Снимок | Николай | целое | Точка × площадка | MVP | На дату снимка; прирост к предыдущему снимку считает система |

### 3.1 Производные

| ID | Показатель | Формула / логика | Зависимости | Статус |
|----|------------|------------------|-------------|--------|
| REP-REV-DELTA | Прирост отзывов между снимками | Разница `REP-REV-CNT` с предыдущим снимком той же пары точка×площадка | REP-REV-CNT | MVP |
| REP-RATING-DELTA | Изменение оценки | Текущий `REP-RATING` минус предыдущий снимок | REP-RATING | MVP |

**Агрегация «месяц/квартал» для репутации:** на дашборде по календарю — брать **последний снимок** в выбранном периоде по каждой паре точка×площадка (зафиксировано в `mvp-scope.md`). Графики по времени внутри периода — по всем снимкам в оси даты; сводная цифра за месяц/квартал — по правилу «последний снимок».

## 4. Ozon (маркетплейс: продажи, возвраты, реклама)

**Смысл для продукта:** Ozon — и **канал продаж**, и **виртуальная точка** с отдельной выручкой; набор полей и логика периода те же, что у офлайн-точек в §1. Период — **отчётная неделя**. **Возвраты не уменьшают `OZ-REV`** — см. правила ведения и §1. Реклама на Ozon — поле `OZ-AD-SPEND` (см. §2.0), в интерфейсе разместить в блоке ввода Ozon.

| OZ-ORD | Заказы Ozon | Первичный | Николай | шт | MVP | |
| OZ-RET-N | Возвраты, количество | Первичный | Николай | шт | MVP | Не влияет на OZ-REV |

## 5. Сводные производные (компания)

Суммы выручки и доля Ozon — из **валовой** выручки по полям `OFF-REV` / `OZ-REV` **без** вычета возвратов (возвраты только отдельными рядами/виджетами).

| ID | Показатель | Формула / логика | Зависимости | Статус |
|----|------------|------------------|-------------|--------|
| DER-REV-TOT | Выручка всего за неделю | Сумма `OFF-REV` по всем офлайн-точкам + `OZ-REV` | OFF-REV, OZ-REV | MVP |
| DER-ORD-TOT | Заказы всего за неделю | Сумма `OFF-ORD` по точкам + `OZ-ORD` | OFF-ORD, OZ-ORD | MVP |
| DER-OZ-SHARE | Доля выручки Ozon | `OZ-REV / DER-REV-TOT` при `DER-REV-TOT > 0` | OZ-REV, DER-REV-TOT | MVP | Доля в «компании за неделю» |
| DER-AD-TOTAL | Расходы на рекламу всего | `MKT-AD-CTX + MKT-AD-MAP + OZ-AD-SPEND` | §2.0, OZ-AD-SPEND | MVP |

**Месяц / квартал:** суммирование **недельных** первичных и пересчёт производных в границах календарного месяца/квартала (для репутации — отдельная логика, §3).
