# Runbook — Grove Pulse (скелет)

**Версия:** 0.1  
**Дата:** 2026-03-31  
**Статус:** заполняется при первом деплое  

Цель: деплой, конфигурация, бэкап, TLS, откат. Детали дополнять по факту настройки VPS; **фактический IP/домен** — в секрет-хранилище или локальной копии, в git — плейсхолдер **`<PROD_VPS_IP>`**.

---

## 1. Окружения

| Окружение | Назначение |
|-----------|------------|
| Localhost | Разработка (Docker Compose по плану репозитория) |
| Production | VPS, публичный IP, HTTPS |

---

## 2. Переменные окружения (черновик)

- Строка подключения **PostgreSQL**
- **Секрет сессии** (подпись/энтропия для cookie — по схеме из [`dev-handoff-spec.md`](./dev-handoff-spec.md) §9)
- **CORS** origin (URL фронта на prod: `https://<PROD_VPS_IP>` или домен)
- Пароли **не** в git — только на сервере и в `.env` локально (`.gitignore`)

Шаблон: **`.env.example`** в корне репозитория (когда появится код).

---

## 3. TLS (HTTPS)

- Требование: [`dev-handoff-spec.md`](./dev-handoff-spec.md) NFR-06.
- Доступ только по **IP**: Let’s Encrypt часто **не** выдаёт сертификат на голый IP — варианты: **самоподписанный** (доверие на устройствах команды), **коммерческий** сертификат на IP, либо **домен** + ACME.

*Заполнить после выбора способа: команды, пути к сертификатам, обновление nginx.*
